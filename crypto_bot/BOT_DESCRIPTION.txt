Папка: c:\Users\Admin\VSC программы\crypto_bot\

Файлы:
- main.py          — точка входа, регистрация хендлеров и ConversationHandler,
                     запуск polling и job_queue, автоперезапуск (run_with_restart)
- config.py        — токен (из .env), список 50 криптовалют (CRYPTO_LIST),
                     список 50 акций (STOCKS_LIST),
                     состояния ConversationHandler (1-40), константы API,
                     FREE_CRYPTOS, PRO_CRYPTOS, FREE_STOCKS, PRO_STOCKS,
                     PROMO_CODES, THRESHOLD_OPTIONS
- crypto_api.py    — работа с CoinGecko API (бесплатный, без ключа),
                     кеширование цен (30 сек TTL), aiohttp
- stocks_api.py    — работа с Yahoo Finance (yfinance),
                     кеширование цен (60 сек TTL), ThreadPoolExecutor,
                     формат данных совместим с CoinGecko
- database.py      — SQLite база (crypto_bot.db), класс Database, 42+ методов
                     Таблицы: users, alerts, tracked_cryptos, tracker_notifications,
                     favorites, subscriptions, used_promos, quiz_rewards
                     Столбец asset_type в alerts, tracked_cryptos,
                     tracker_notifications, favorites (crypto/stocks)
- handlers.py      — ~6800+ строк, все хендлеры: 150+ функций,
                     двухрежимная система (crypto/stocks),
                     хелперы: get_mode, get_asset_list, get_allowed_assets,
                     asset_label, fetch_asset_prices и др.
- requirements.txt — зависимости: python-telegram-bot[job-queue]==22.6,
                     python-dotenv, aiohttp, yfinance
- .env             — BOT_TOKEN (уже заполнен)
- BOT_DESCRIPTION.txt — ЭТО ОПИСАНИЕ (обновляй после изменений!)

================================================================================
                        БАЗА ДАННЫХ (SQLite)
================================================================================

Файл: crypto_bot.db

ТАБЛИЦЫ:
1. users — пользователи
   (user_id, username, created_at)

2. alerts — ценовые алерты
   (alert_id, user_id, crypto_ticker, target_price, currency, direction, active, triggered, created_at)

3. tracked_cryptos — отслеживаемые крипты (трекер)
   (user_id, crypto_ticker, threshold, added_at)

4. tracker_notifications — кулдаун уведомлений трекера
   (user_id, crypto_ticker, direction, last_notified)

5. favorites — избранные крипты
   (user_id, crypto_ticker, added_at)

6. subscriptions — подписки пользователей
   (user_id, tier [free/pro/premium], expires_at [NULL=навсегда], created_at)

7. used_promos — использованные промокоды
   (user_id, promo_code, used_at)

8. quiz_rewards — попытки и награды викторины
   (user_id, difficulty, last_attempt, last_reward)

СВЯЗИ:
- Все таблицы связаны через user_id
- alerts: UNIQUE(user_id, crypto_ticker, currency, direction, target_price)
- tracked_cryptos: UNIQUE(user_id, crypto_ticker)
- favorites: UNIQUE(user_id, crypto_ticker)
- subscriptions: PRIMARY KEY(user_id)
- used_promos: UNIQUE(user_id, promo_code)
- quiz_rewards: UNIQUE(user_id, difficulty)

МЕТОДЫ DATABASE (42 метода):
- Пользователи: add_user, get_user
- Алерты: create_alert, get_user_alerts, get_all_active_alerts, trigger_alert,
  delete_alert, delete_all_user_alerts, deactivate_alert, count_user_active_alerts
- Трекер: add_tracked_crypto, remove_tracked_crypto, get_user_tracked, get_all_tracked,
  set_tracked_threshold, set_all_tracked_threshold, count_user_tracked, is_tracked,
  can_notify_tracker, update_tracker_notification, clear_user_tracked
- Избранное: add_favorite, remove_favorite, get_user_favorites, is_favorite,
  count_favorites, clear_favorites
- Подписки: get_subscription, get_active_tier, set_subscription, add_subscription_days
- Промокоды: is_promo_used, use_promo
- Викторина: get_quiz_rewards, set_quiz_attempt, set_quiz_reward,
  can_get_quiz_reward, can_attempt_quiz_reward

1. check_alerts_job — проверка алертов каждые 60 секунд
   - Загружает все активные алерты → проверяет текущие цены
   - При срабатывании → уведомление пользователю + деактивация алерта

2. check_tracker_job — проверка трекера каждые 120 секунд
   - Загружает все отслеживаемые крипты → сравнивает с порогами
   - Кулдаун 4 часа (TRACKER_COOLDOWN = 14400 сек)
   - При срабатывании → уведомление с деталями изменения

3. check_predictions_job — проверка предсказаний каждые 300 секунд
   - Проверяет предсказания старше 24 часов
   - Сравнивает с текущей ценой → уведомление о результате
