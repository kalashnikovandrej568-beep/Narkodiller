📌 ЧТО ЭТО?
Это Telegram-бот с полноценной системой ставок на события. Бот позволяет
пользователям делать ставки виртуальной валютой (монетами) на различные
события. Есть ДВА типа событий:
1) Автоматические — бот сам создаёт и закрывает события каждые 5 минут
2) Админские — события созданные администратором вручную

📌 НАЗНАЧЕНИЕ
Бот создан для развлекательных ставок в Telegram-чатах и группах. Он имитирует
букмекерскую систему с виртуальной валютой: пользователи получают стартовый
баланс и могут делать ставки на события (спортивные матчи, поединки, любые
пользовательские события). Реальные деньги НЕ используются — только внутренняя
валюта "монеты".

1. РЕГИСТРАЦИЯ ПОЛЬЗОВАТЕЛЕЙ
   - При первом /start пользователь автоматически регистрируется
   - Получает стартовый баланс: 100 монет
   - Данные хранятся в таблице users (user_id, username, balance, wins, losses)

2. СИСТЕМА СТАВОК
   - Админ создаёт событие с участниками и их силой (1-10)
   - Система автоматически рассчитывает шансы и коэффициенты
   - Формула: шанс = сила_участника / сумма_всех_сил × 100%
   - Формула коэффициента: коэффициент = сумма_всех_сил / сила_участника
   - Пользователь выбирает событие → вводит сумму → выбирает участника
   - Минимальная ставка: 10 монет, максимум — весь баланс
   - Одна ставка на событие (защита от дублей)

3. ОПРЕДЕЛЕНИЕ ПОБЕДИТЕЛЯ
   - При закрытии события победитель выбирается АВТОМАТИЧЕСКИ
   - Используется random.choices() с весами на основе силы участников
   - Чем больше сила — тем выше шанс победы, но ниже коэффициент выплаты
   - Выигрыш = ставка × коэффициент победителя

4. МИНИ-ИГРА: РУЛЕТКА (🎰)
   - Пользователь выбирает цвет: красное, чёрное или зелёное
   - Числа 0-36: 0 = зелёное, нечётное = красное, чётное = чёрное
   - Красное/Чёрное: выплата x2 (шанс 18/37 ≈ 48.6%)
   - Зелёное: выплата x10 (шанс 1/37 ≈ 2.7%)
   - Flow: кнопка "🎰 Рулетка" → ввод суммы → выбор цвета → мгновенный результат
   - Результат записывается в game_results и обновляет wins/losses

5. МИНИ-ИГРА: МОНЕТКА (🪙)
   - Классическая игра: орёл или решка, 50/50 шанс
   - Выплата x2 при угадывании
   - Flow: кнопка "🪙 Монетка" → ввод суммы → выбор стороны → мгновенный результат
   - Результат записывается в game_results и обновляет wins/losses

6. МИНИ-ИГРА: КОСТИ (🎯 Dice Duel)
   - Игрок и бот бросают по 2 кубика (1-6 каждый)
   - У кого сумма больше — тот победил
   - Выплата x2 при победе, x3 при победе с дублем (оба кубика одинаковые)
   - Ничья (одинаковые суммы) = возврат ставки
   - Визуализация: символы кубиков ⚀⚁⚂⚃⚄⚅
   - Flow: кнопка "🎯 Кости" → ввод суммы → автоматический бросок → результат
   - Результат записывается в game_results и обновляет wins/losses

7. МИНИ-ИГРА: БЛЭКДЖЕК (🃏)
   - Упрощённый блэкджек против бота-дилера
   - Карты: 2-10 = номинал, J/Q/K = 10, Туз = 11 или 1
   - Колода: 52 карты (4 масти × 13 номиналов)
   - Блэкджек (21 с 2 карт) = выплата x2.5
   - Обычная победа = x2, Ничья = возврат, Перебор (>21) = проигрыш
   - Дилер берёт карты до 17
   - Визуализация: масти ♠️ ♥️ ♦️ ♣️, скрытая карта дилера 🂠
   - Flow: кнопка "🃏 Блэкджек" → ввод суммы → раздача → Ещё/Хватит → результат
   - Результат записывается в game_results и обновляет wins/losses

8. МИНИ-ИГРА: СЛОТЫ (🍀)
   - Слот-машина с 3 барабанами и 8 символами
   - Символы: 🍒 🍋 🍊 🍇 🔔 ⭐ 💎 7️⃣
   - 3 одинаковых = выигрыш с множителем (от x2 до x20)
   - 2 одинаковых = x1.5 (возврат с бонусом)
   - Без совпадений = проигрыш
   - Множители: 7️⃣x20, 💎x15, ⭐x10, 🔔x7, 🍇x5, 🍊x4, 🍋x3, 🍒x2
   - Flow: кнопка "🍀 Слоты" → ввод суммы → мгновенная крутка → результат
   - Визуализация рамкой ╔═╗/╚═╝
   - Результат записывается в game_results и обновляет wins/losses

8. БАНКРОТ-ВОССТАНОВЛЕНИЕ (💊)
   - Если баланс пользователя == 0, доступна кнопка "💊 Банкрот"
   - Даёт 50 монет, доступно 1 раз в день (отдельно от дневного бонуса)
   - Отслеживается в таблице bankrupt_recovery

7. ДНЕВНОЙ БОНУС

   - Дата последнего бонуса хранится в таблице daily_bonuses

5. ТАБЛИЦА ЛИДЕРОВ

6. ИСТОРИЯ СТАВОК

8. АВТОМАТИЧЕСКИЕ СОБЫТИЯ (🤖)

================================================================================
                           БАЗА ДАННЫХ (SQLite)
================================================================================

ТАБЛИЦЫ:
1. users — пользователи (user_id, username, balance, total_wins, total_losses, created_at)
2. events — события (event_id, title, description, odds, status, winner, participants, strengths, created_at, closed_at)
3. bets — ставки (bet_id, user_id, event_id, amount, status, result, created_at)
4. authorized_admins — авторизованные админы (user_id, username, added_at)
5. daily_bonuses — дневные бонусы (user_id, last_bonus_date)
6. game_results — результаты мини-игр (game_id, user_id, game_type, bet_amount, won, winnings, created_at)
7. bankrupt_recovery — банкрот-восстановление (user_id, last_recovery_date)

СВЯЗИ:
- bets.user_id → users.user_id
- bets.event_id → events.event_id
- participants и strengths хранятся как JSON-строки в events
